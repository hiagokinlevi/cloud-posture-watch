# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.x     | Yes       |

## Responsible disclosure

If you discover a security vulnerability in `cloud-posture-watch`, please **do not** open a public GitHub issue. Instead, report it privately so we can address it before public disclosure.

**Report via:** GitHub's private vulnerability reporting feature (Security > Report a vulnerability on this repository).

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof-of-concept
- The version(s) affected
- Any suggested mitigations (optional)

We aim to acknowledge reports within **72 hours** and provide an initial assessment within **7 days**.

## Scope

This tool is designed to **read** cloud configurations. It never stores or transmits credentials. However, if you discover a way that running this tool could unintentionally expose credentials or allow privilege escalation, that is in scope.

## Out of scope

- Findings generated by the tool against your own cloud environment (those are working as intended)
- Bugs that require physical access to a machine running the tool

## Disclosure timeline

Once a fix is ready we will:

1. Release a patched version
2. Publish a GitHub Security Advisory
3. Credit the reporter (unless anonymity is requested)
